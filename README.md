# Logistic Agent
Интеллектуальная система поддержки принятия решений для логистических цепочек. Агент анализирует неструктурированные отчеты (PDF/Новости) и сопоставляет их с оперативными данными из SQL-базы для выявления рисков и задержек.

## Основные возможности
- Advanced RAG: Использование LlamaParse для сохранения структуры таблиц из сложных PDF-отчетов.
- Hybrid Search: Объединение семантического поиска по векторному индексу (Qdrant) и структурированных запросов к реляционной БД (PostgreSQL).
- Agentic Workflow: Событийно-ориентированная архитектура на базе LlamaIndex Workflows с этапами маршрутизации (Routing) и самопроверки (Reflection).
- Observability: Полный трекинг токенов и логирование цепочки рассуждений.

## Стек технологий
- Orchestration: LlamaIndex (Workflows)
- LLM: DeepSeek-V3
- Vector DB: Qdrant Cloud
- SQL DB: PostgreSQL
- Parsing: LlamaParse
- Language: Python 3.10+

## Логика работы (Workflow)
Агент работает по циклической схеме:
1. Input: Получение запроса от пользователя.
2. Routing: Определение необходимых инструментов (SQL для цифр, Vector для контекста).
3. Retrieval: Параллельный сбор данных из облачных БД.
4. Reflection: Проверка ответа на галлюцинации и соответствие фактам.
5. Output: Выдача итогового решения.
